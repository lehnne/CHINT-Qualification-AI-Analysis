# 员工能力标签写入接口 · 数据中台对接文档

> 数据中台将 Agent 输出的标签 JSON 写入 PostgreSQL 的 `employee_capability_tags` 表。
> 核心难点：输入是嵌套JSON（维度→标签数组），需要展平为数据库行，再 UPSERT。

## 二、核心逻辑：JSON 展平

输入是嵌套结构，数据库是扁平行，需要做一次**维度展开**：

```
输入 JSON：
  employee_id: "EMP_12345"
  dimension_tags: {
    "技术专长": [tag1, tag2],       ← 2个标签
    "技术创新": [tag3]               ← 1个标签
  }

展平后 = 3 行：
  (EMP_12345, 2025-H1, 技术专长, 复杂架构设计, 88, ...)
  (EMP_12345, 2025-H1, 技术专长, 高并发优化, 85, ...)
  (EMP_12345, 2025-H1, 技术创新, 工具/流程创新, 78, ...)
```

## 三、PostgreSQL 函数（推荐方案）

写一个 **PostgreSQL 函数**，接收整个 JSON 请求体，内部完成展平 + UPSERT。

```sql
-- ============================================================
-- 函数：upsert_employee_tags
-- 功能：接收 Agent 输出的标签 JSON，展平并 UPSERT 到 employee_capability_tags
-- 输入：p_payload JSONB — 完整的 PUT 请求体
-- 返回：写入统计（总员工数、总标签数、新增数、更新数）
-- 用法：SELECT * FROM upsert_employee_tags('{...}'::JSONB);
-- ============================================================
CREATE OR REPLACE FUNCTION upsert_employee_tags(p_payload JSONB)
RETURNS TABLE (
    batch_id         VARCHAR(50),
    total_employees  INT,
    total_tags       INT,
    inserted         INT,
    updated          INT
) AS $$
DECLARE
    v_batch_id        VARCHAR(50);
    v_assessment_cycle VARCHAR(20);
    v_mode             VARCHAR(20);
    v_emp_rec          RECORD;
    v_dim_key          TEXT;
    v_tag_rec          RECORD;
    v_inserted         INT := 0;
    v_updated          INT := 0;
    v_total_tags       INT := 0;
    v_total_emps       INT := 0;
BEGIN
    -- 提取公共字段
    v_batch_id        := p_payload ->> 'batch_id';
    v_assessment_cycle := p_payload ->> 'assessment_cycle';
    v_mode             := p_payload ->> 'mode';

    -- mode = 'initial' 时，先清空该周期的旧数据（首次全量初始化）
    IF v_mode = 'initial' THEN
        DELETE FROM employee_capability_tags
        WHERE assessment_cycle = v_assessment_cycle;
    END IF;

    -- 遍历 tags 数组
    FOR v_emp_rec IN
        SELECT * FROM JSONB_TO_RECORDSET(p_payload -> 'tags')
          AS x(
              employee_id          VARCHAR(50),
              employee_name        VARCHAR(50),
              position_family_code VARCHAR(20),
              position_family_name VARCHAR(50),
              original_position    VARCHAR(100),
              original_grade       VARCHAR(20),
              target_grade         VARCHAR(20),
              dimension_tags       JSONB
          )
    LOOP
        v_total_emps := v_total_emps + 1;

        -- 遍历 dimension_tags 的每个维度
        FOR v_dim_key IN
            SELECT JSONB_OBJECT_KEYS(v_emp_rec.dimension_tags)
        LOOP
            -- 遍历该维度下的标签数组
            FOR v_tag_rec IN
                SELECT * FROM JSONB_TO_RECORDSET(
                    v_emp_rec.dimension_tags -> v_dim_key
                ) AS y(
                    tag_name         VARCHAR(100),
                    score            SMALLINT,
                    confidence       DECIMAL(3,2),
                    evidence         TEXT,
                    source_materials JSONB
                )
            LOOP
                v_total_tags := v_total_tags + 1;

                -- UPSERT：有则更新，无则插入
                INSERT INTO employee_capability_tags (
                    employee_id,          employee_name,
                    position_family_code, position_family_name,
                    original_position,    original_grade,
                    target_grade,         assessment_cycle,
                    dimension,            tag_name,
                    score,                confidence,
                    evidence,             source_materials,
                    batch_id,             created_at,
                    updated_at
                ) VALUES (
                    v_emp_rec.employee_id,          v_emp_rec.employee_name,
                    v_emp_rec.position_family_code, v_emp_rec.position_family_name,
                    v_emp_rec.original_position,    v_emp_rec.original_grade,
                    v_emp_rec.target_grade,         v_assessment_cycle,
                    v_dim_key,                      v_tag_rec.tag_name,
                    v_tag_rec.score,                v_tag_rec.confidence,
                    v_tag_rec.evidence,             v_tag_rec.source_materials,
                    v_batch_id,                     NOW(),
                    NOW()
                )
                ON CONFLICT (employee_id, assessment_cycle, dimension, tag_name)
                DO UPDATE SET
                    score             = EXCLUDED.score,
                    confidence        = EXCLUDED.confidence,
                    evidence          = EXCLUDED.evidence,
                    source_materials  = EXCLUDED.source_materials,
                    original_grade    = EXCLUDED.original_grade,
                    target_grade      = EXCLUDED.target_grade,
                    batch_id          = EXCLUDED.batch_id,
                    updated_at        = NOW();

                -- 统计影响行数
                IF FOUND THEN
                    IF (SELECT COUNT(*) FROM employee_capability_tags
                        WHERE employee_id = v_emp_rec.employee_id
                          AND assessment_cycle = v_assessment_cycle
                          AND dimension = v_dim_key
                          AND tag_name = v_tag_rec.tag_name
                          AND created_at = updated_at) > 0 THEN
                        v_inserted := v_inserted + 1;
                    ELSE
                        v_updated := v_updated + 1;
                    END IF;
                END IF;
            END LOOP;
        END LOOP;
    END LOOP;

    -- 返回统计结果
    RETURN QUERY
    SELECT
        v_batch_id,
        v_total_emps,
        v_total_tags,
        v_inserted,
        v_updated;
END;
$$ LANGUAGE plpgsql;
```

## 五、唯一约束说明

**当前表上的唯一约束是**：

```sql
UNIQUE (employee_id, assessment_cycle, dimension, tag_name)
```

**含义**：同一人、同一评审周期、同一维度下，不能有同名的标签。

**如果重复写入会怎样？**
- 不会报错，函数里用的是 `ON CONFLICT ... DO UPDATE`，会覆盖更新
- 所以幂等性是有保证的——同样的数据写两次，结果一样

## 六、数据中台 API 包装建议

数据中台收到 HTTP PUT 请求后的处理流程：

```
HTTP PUT /api/v1/tags/employee
  │
  ├─ 1. 校验请求体格式（必填字段检查）
  ├─ 2. 调用函数：SELECT * FROM upsert_employee_tags(p_payload)
  ├─ 3. 包装返回：
  │     {
  │       "code": 0,
  │       "message": "success",
  │       "data": { 函数返回结果 }
  │     }
  └─ 4. 异常时返回：
        {
          "code": 40001,
          "message": "参数校验失败：缺少必填字段 batch_id",
          "data": null
        }
```

### 必填字段校验清单

| 字段路径 | 必填 | 说明 |
|---------|------|------|
| `batch_id` | 是 | 批次ID，非空 |
| `assessment_cycle` | 是 | 评审周期，非空 |
| `mode` | 是 | 只能是 `initial` 或 `incremental` |
| `tags[].employee_id` | 是 | 员工工号，非空 |
| `tags[].employee_name` | 是 | 员工姓名，非空 |
| `tags[].position_family_code` | 是 | 岗位序列代码，非空 |
| `tags[].position_family_name` | 是 | 岗位序列名称，非空 |
| `tags[].dimension_tags` | 是 | 维度标签对象，不能为空对象 |
| `dimension_tags[].tag_name` | 是 | 标签名称，非空 |
| `dimension_tags[].score` | 是 | 0-100 整数 |

## 八、常见问题

### Q1：为什么不用单独 INSERT 每条，而要用函数？

函数一次性处理整个 JSON，数据中台只需要调用一次，不需要在应用层写循环逻辑。如果不用函数，数据中台要先解析 JSON → 按维度展开 → 逐条 INSERT，代码更复杂。

### Q2：mode = 'initial' 会清空旧数据，误操作怎么办？

initial 模式只会在系统上线第一天使用。日常跑的都是 incremental 模式。如果担心误操作，可以在函数内部加一个保护逻辑：

```sql
-- 只允许在凌晨 2:00-4:00 之间执行 initial 模式
IF v_mode = 'initial' AND EXTRACT(HOUR FROM NOW()) NOT BETWEEN 2 AND 4 THEN
    RAISE EXCEPTION 'initial 模式只能在凌晨 2:00-4:00 之间执行';
END IF;
```

### Q3：如果不创建函数，数据中台直接用 SQL 怎么写？

直接使用 UPSERT SQL，但需要先展平 JSON：

```sql
-- 先用 JSONB_TO_RECORDSET 展开 dimension_tags
-- 需要数据中台在应用层做循环，逐条调用以下 UPSERT：
INSERT INTO employee_capability_tags (...)
VALUES (...)
ON CONFLICT (employee_id, assessment_cycle, dimension, tag_name)
DO UPDATE SET ...;
```
